## Descripción del Proyecto

Implementación de un **algoritmo genético paralelizado en CUDA** para resolver una versión extendida del problema de la mochila (knapsack).

## Restricciones (Duras vs Blandas)

### Restricciones DURAS (solución final DEBE cumplir):
- Peso máximo W (≤ 40% del peso total)
- Volumen máximo V (≤ 40% del volumen total)
- Incompatibilidades entre ítems
- Dependencias obligatorias

### Restricciones BLANDAS (penalizadas en fitness, permiten violaciones temporales):
- Restricciones por categoría (min/máx)

## Función de Aptitud (Fitness)

```
fitness(X) = Σ(vi * xi) - α*exceso_peso - β*exceso_volumen - γ*violaciones_categoria - δ*incompatibilidades - ε*dependencias_incumplidas
```

Los valores de α, β, γ, δ, ε deben definirse y justificarse.

## 3 Variantes Obligatorias

### Variante 1: CPU Secuencial (baseline)
- Generación población inicial
- Evaluación aptitud
- Selección por torneo
- Cruzamiento (uniforme o de un punto)
- Mutación (flip bits)
- Elitismo (preservar mejor individuo)
- Criterio término: N generaciones o convergencia

### Variante 2: CUDA Básico
Paralelizar al menos:
- Evaluación fitness (1 hilo/individuo)
- Selección por torneo
- Cruzamiento y mutación
- Actualización generación
- Registro mejor individuo factible
- Población permanece en GPU entre generaciones

### Variante 3: CUDA Optimizado (≥5 estrategias)
- Shared memory (reutilización en bloque)
- Memoria constante (parámetros, solo lectura)
- Accesos coalescentes
- Reducciones paralelas (fitness, pesos, mejores)
- Menos transferencias host↔device
- Ajuste block size / grid size
- Streams CUDA
- Control divergencia warps
- RNG por hilo/individuo/generación

## Resultados Finales (2026-06-10)

| Instancia | Items | Incompat | CPU (ms) | CUDA Básico (ms) | CUDA Opt (ms) | Speed-up | Factible |
|---|---|---|---|---|---|---|---|
| Small | 100 | 99 | 1,861 | 106 | 104 | ~18x | 56.5% |
| Medium | 1,000 | 99 | 35,389 | 981 | 921 | ~38x | 36.0% |

**Configuración de penalizaciones:**
- α=100×scale (peso), β=100×scale (volumen), γ=50×scale (categoría)
- δ=1,000,000 (incompatibilidad = "death penalty"), ε=1,000,000 (dependencia)
- scale = n_items / 100

**Hallazgos técnicos clave:**
- Driver 591.86 requiere `-arch=sm_86` (genera SASS, no PTX)
- CUDA 13.3 toolkit funciona con driver CUDA 13.1
- Inicialización greedy (25% de capacidad) mejora factibilidad vs aleatorio puro
- Elitismo: copiar top-4 directamente preservan las mejores soluciones
- Para medium/large, incompatibilidades deben ser <0.02% de pares posibles

## Compilación (verificado funcional)

```bash
# Desde bash (MSYS2/Git Bash)
cd Documents/software3-cuda
CUDA_BIN="/c/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.3/bin"
CUDA_INC="/c/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.3/include"
CUDA_LIB="/c/Program Files/NVIDIA GPU Computing Toolkit/CUDA/v13.3/lib/x64"
MSVC_BIN="/c/Program Files (x86)/Microsoft Visual Studio/2022/BuildTools/VC/Tools/MSVC/14.44.35207/bin/Hostx64/x64"
MSVC_INC="/c/Program Files (x86)/Microsoft Visual Studio/2022/BuildTools/VC/Tools/MSVC/14.44.35207/include"
MSVC_LIB="/c/Program Files (x86)/Microsoft Visual Studio/2022/BuildTools/VC/Tools/MSVC/14.44.35207/lib/x64"
SDK_VER="10.0.26100.0"
SDK_BASE="/c/Program Files (x86)/Windows Kits/10"

export PATH="$CUDA_BIN:$MSVC_BIN:$PATH"

nvcc -O3 -std=c++17 -arch=sm_86 \
  -I"$CUDA_INC" -I"$MSVC_INC" \
  -I"$SDK_BASE/Include/$SDK_VER/um" -I"$SDK_BASE/Include/$SDK_VER/ucrt" -I"$SDK_BASE/Include/$SDK_VER/shared" \
  -Isrc \
  -L"$CUDA_LIB" -L"$MSVC_LIB" -L"$SDK_BASE/Lib/$SDK_VER/um/x64" -L"$SDK_BASE/Lib/$SDK_VER/ucrt/x64" \
  -o mochila_ga_cuda src/*.cu src/*.cpp
```
